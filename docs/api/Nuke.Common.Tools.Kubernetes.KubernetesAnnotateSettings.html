Class KubernetesAnnotateSettings

Namespace
Nuke.Common.Tools.Kubernetes
Assembly
Nuke.Common.dll

Update the annotations on one or more resources All Kubernetes objects support the ability to store additional data with the object as annotations. Annotations are key/value pairs that can be larger than labels and include arbitrary string values such as structured JSON. Tools and system extensions may use annotations to store their own data. Attempting to set an annotation that already exists will fail unless --overwrite is set. If --resource-version is specified and does not match the current resource version on the server the command will fail.Use "kubectl api-resources" for a complete list of supported resources.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <annotations> via Annotations
  • <typeName> via TypeName
  • --all via All
  • --allow-missing-template-keys via AllowMissingTemplateKeys
  • --alsologtostderr via Alsologtostderr
  • --as via As
  • --as-group via AsGroup
  • --cache-dir via CacheDir
  • --certificate-authority via CertificateAuthority
  • --client-certificate via ClientCertificate
  • --client-key via ClientKey
  • --cluster via Cluster
  • --context via Context
  • --dry-run via DryRun
  • --field-selector via FieldSelector
  • --filename via Filename
  • --include-uninitialized via IncludeUninitialized
  • --insecure-skip-tls-verify via InsecureSkipTlsVerify
  • --kubeconfig via Kubeconfig
  • --local via Local
  • --log-backtrace-at via LogBacktraceAt
  • --log-dir via LogDir
  • --logtostderr via Logtostderr
  • --match-server-version via MatchServerVersion
  • --namespace via Namespace
  • --output via Output
  • --overwrite via Overwrite
  • --record via Record
  • --recursive via Recursive
  • --request-timeout via RequestTimeout
  • --resource-version via ResourceVersion
  • --selector via Selector
  • --server via Server
  • --stderrthreshold via Stderrthreshold
  • --template via Template
  • --token via Token
  • --user via User
  • --v via V
  • --vmodule via Vmodule
