One of ReSharper's most notable features, code inspection, is available even without opening Visual Studio. InspectCode, a free command line tool requires a minimum of one parameter- your solution file- to apply all of ReSharper's inspections.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <targetPath> via TargetPath
  • --absolute-paths via AbsolutePaths
  • --caches-home via CachesHome
  • --config via ConfigFile
  • --config-create via CreateConfigFile
  • --debug via Debug
  • --disable-settings-layers via DisableSettingsLayers
  • --dotnetcore via DotNetCore
  • --dotnetcoresdk via DotNetCoreSdk
  • --dumpIssuesTypes via DumpIssuesTypes
  • --exclude via Exclude
  • --format via Format
  • --include via Include
  • --jobs via Jobs
  • --mono via MonoPath
  • --no-buildin-settings via NoBuiltinSettings
  • --no-swea via NoSwea
  • --output via Output
  • --project via Project
  • --properties via Properties
  • --settings via Settings
  • --severity via Severity
  • --targets-for-items via ItemTargets
  • --targets-for-references via ReferenceTargets
  • --toolset via Toolset
  • --toolset-path via ToolsetPath
  • --verbosity via Verbosity

AddProperty<T>(T, string, string)

Lets you override MSBuild properties. You can set each property separately ( --properties:prop1=val1 --properties:prop2=val2), or use a semicolon to separate multiple properties --properties:prop1=val1;prop2=val2.

Note that the semicolon cannot be used inside values, for example: --properties:ReferencePath="r:\reference1\;r:\reference2\". In such cases, add each value separately using another --properties parameter — the values will be combined.

The specified properties are applied to all analyzed projects. Currently, there is no direct way to set a property to a specific project only. The workaround is to create a custom property in this project and assign it to the desired property, then use the custom property in CleanupCode parameters.

Note that the semicolon cannot be used inside values, for example: --properties:ReferencePath="r:\reference1\;r:\reference2\". In such cases, add each value separately using another --properties parameter — the values will be combined.

The specified properties are applied to all analyzed projects. Currently, there is no direct way to set a property to a specific project only.
