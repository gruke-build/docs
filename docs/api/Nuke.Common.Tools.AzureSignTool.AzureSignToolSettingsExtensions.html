Class AzureSignToolSettingsExtensions

Namespace
Nuke.Common.Tools.AzureSignTool
Assembly
Nuke.Common.dll

Azure Sign Tool is similar to signtool in the Windows SDK, with the major difference being that it uses Azure Key Vault for performing the signing process. The usage is like signtool, except with a limited set of options for signing and options for authenticating to Azure Key Vault.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <files> via Files
  • --additional-certificates via AdditionalCertificates
  • --azure-key-vault-accesstoken via KeyVaultAccessToken
  • --azure-key-vault-certificate via KeyVaultCertificateName
  • --azure-key-vault-client-id via KeyVaultClientId
  • --azure-key-vault-client-secret via KeyVaultClientSecret
  • --azure-key-vault-managed-identity via KeyVaultManagedIdentity
  • --azure-key-vault-tenant-id via KeyVaultTenantId
  • --azure-key-vault-url via KeyVaultUrl
  • --continue-on-error via ContinueOnError
  • --description via Description
  • --description-url via DescriptionUrl
  • --file-digest via FileDigest
  • --input-file-list via InputFileList
  • --max-degree-of-parallelism via MaxDegreeOfParallelism
  • --no-page-hashing via NoPageHashing
  • --page-hashing via PageHashing
  • --quiet via Quiet
  • --skip-signed via SkipSigned
  • --timestamp-authenticode via TimestampAuthenticodeUrl
  • --timestamp-digest via TimestampDigest
  • --timestamp-rfc3161 via TimestampRfc3161Url
  • --verbose via Verbose

Methods

AddAdditionalCertificates<T>(T, IEnumerable<string>)

A list of paths to additional certificates to aide in building a full chain for the signing certificate. Azure SignTool will build a chain, either as deep as it can or to a trusted root. This will also use the Windows certificate store, in addition to any certificates specified with this option. Specifying this option does not guarantee the inclusion of the certificate, only if it is part of the chain. The files specified must be public certificates only. They cannot be PFX, PKCS12 or PFX files.
