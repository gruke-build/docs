Class HelmRollbackSettingsExtensions

Namespace
Nuke.Common.Tools.Helm
Assembly
Nuke.Common.dll

This command rolls back a release to a previous revision. The first argument of the rollback command is the name of a release, and the second is a revision (version) number. To see revision numbers, run 'helm history RELEASE'. If you'd like to rollback to the previous release use 'helm rollback [RELEASE] 0'.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <release> via Release
  • <revision> via Revision
  • --debug via Debug
  • --description via Description
  • --dry-run via DryRun
  • --force via Force
  • --home via Home
  • --host via Host
  • --kube-context via KubeContext
  • --kubeconfig via Kubeconfig
  • --no-hooks via NoHooks
  • --recreate-pods via RecreatePods
  • --tiller-connection-timeout via TillerConnectionTimeout
  • --tiller-namespace via TillerNamespace
  • --timeout via Timeout
  • --tls via Tls
  • --tls-ca-cert via TlsCaCert
  • --tls-cert via TlsCert
  • --tls-hostname via TlsHostname
  • --tls-key via TlsKey
  • --tls-verify via TlsVerify
  • --wait via Wait
