Class KubernetesLabelSettings

Namespace
Nuke.Common.Tools.Kubernetes
Assembly
Nuke.Common.dll

Update the labels on a resource. * A label key and value must begin with a letter or number, and may contain letters, numbers, hyphens, dots, and underscores, up to 63 characters each. * Optionally, the key can begin with a DNS subdomain prefix and a single '/', like example.com/my-app * If --overwrite is true, then existing labels can be overwritten, otherwise attempting to overwrite a label will result in an error. * If --resource-version is specified, then updates will use this resource version, otherwise the existing resource-version will be used.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <labels> via Labels
  • <typeName> via TypeName
  • --all via All
  • --allow-missing-template-keys via AllowMissingTemplateKeys
  • --alsologtostderr via Alsologtostderr
  • --as via As
  • --as-group via AsGroup
  • --cache-dir via CacheDir
  • --certificate-authority via CertificateAuthority
  • --client-certificate via ClientCertificate
  • --client-key via ClientKey
  • --cluster via Cluster
  • --context via Context
  • --dry-run via DryRun
  • --field-selector via FieldSelector
  • --filename via Filename
  • --include-uninitialized via IncludeUninitialized
  • --insecure-skip-tls-verify via InsecureSkipTlsVerify
  • --kubeconfig via Kubeconfig
  • --list via List
  • --local via Local
  • --log-backtrace-at via LogBacktraceAt
  • --log-dir via LogDir
  • --logtostderr via Logtostderr
  • --match-server-version via MatchServerVersion
  • --namespace via Namespace
  • --output via Output
  • --overwrite via Overwrite
  • --record via Record
  • --recursive via Recursive
  • --request-timeout via RequestTimeout
  • --resource-version via ResourceVersion
  • --selector via Selector
  • --server via Server
  • --stderrthreshold via Stderrthreshold
  • --template via Template
  • --token via Token
  • --user via User
  • --v via V
  • --vmodule via Vmodule
