Class KubernetesExposeSettings

Namespace
Nuke.Common.Tools.Kubernetes
Assembly
Nuke.Common.dll

Expose a resource as a new Kubernetes service. Looks up a deployment, service, replica set, replication controller or pod by name and uses the selector for that resource as the selector for a new service on the specified port. A deployment or replica set will be exposed as a service only if its selector is convertible to a selector that service supports, i.e. when the selector contains only the matchLabels component. Note that if no port is specified via --port and the exposed resource has multiple ports, all will be re-used by the new service. Also if no labels are specified, the new service will re-use the labels from the resource it exposes. Possible resources include (case insensitive): pod (po), service (svc), replicationcontroller (rc), deployment (deploy), replicaset (rs).

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • --allow-missing-template-keys via AllowMissingTemplateKeys
  • --alsologtostderr via Alsologtostderr
  • --as via As
  • --as-group via AsGroup
  • --cache-dir via CacheDir
  • --certificate-authority via CertificateAuthority
  • --client-certificate via ClientCertificate
  • --client-key via ClientKey
  • --cluster via Cluster
  • --cluster-ip via ClusterIp
  • --container-port via ContainerPort
  • --context via Context
  • --dry-run via DryRun
  • --external-ip via ExternalIp
  • --filename via Filename
  • --generator via Generator
  • --insecure-skip-tls-verify via InsecureSkipTlsVerify
  • --kubeconfig via Kubeconfig
  • --labels via Labels
  • --load-balancer-ip via LoadBalancerIp
  • --log-backtrace-at via LogBacktraceAt
  • --log-dir via LogDir
  • --logtostderr via Logtostderr
  • --match-server-version via MatchServerVersion
  • --name via Name
  • --namespace via Namespace
  • --output via Output
  • --overrides via Overrides
  • --port via Port
  • --protocol via Protocol
  • --record via Record
  • --recursive via Recursive
  • --request-timeout via RequestTimeout
  • --save-config via SaveConfig
  • --selector via Selector
  • --server via Server
  • --session-affinity via SessionAffinity
  • --stderrthreshold via Stderrthreshold
  • --target-port via TargetPort
  • --template via Template
  • --token via Token
  • --type via Type
  • --user via User
  • --v via V
  • --vmodule via Vmodule
