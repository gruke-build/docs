Class HelmTemplateSettingsExtensions

Namespace
Nuke.Common.Tools.Helm
Assembly
Nuke.Common.dll

Render chart templates locally and display the output. This does not require Tiller. However, any values that would normally be looked up or retrieved in-cluster will be faked locally. Additionally, none of the server-side testing of chart validity (e.g. whether an API is supported) is done. To render just one template in a chart, use '-x': $ helm template mychart -x templates/deployment.yaml.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <chart> via Chart
  • --debug via Debug
  • --execute via Execute
  • --home via Home
  • --host via Host
  • --is-upgrade via IsUpgrade
  • --kube-context via KubeContext
  • --kube-version via KubeVersion
  • --kubeconfig via Kubeconfig
  • --name via Name
  • --name-template via NameTemplate
  • --namespace via Namespace
  • --notes via Notes
  • --output-dir via OutputDir
  • --set via Set
  • --set-file via SetFile
  • --set-string via SetString
  • --tiller-connection-timeout via TillerConnectionTimeout
  • --tiller-namespace via TillerNamespace
  • --values via Values
