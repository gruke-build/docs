Class SignToolSettings

Namespace
Nuke.Common.Tools.SignTool
Assembly
Nuke.Common.dll

Use the sign command to sign files using embedded signatures. Signing protects a file from tampering, and allows users to verify the signer (you) based on a signing certificate. The options below allow you to specify signing parameters and to select the signing certificate you wish to use.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <files> via Files
  • /a via AutomaticSelection
  • /ac via AdditionalCertificate
  • /as via AppendSignature
  • /c via CertificateTemplateName
  • /csp via Csp
  • /d via Description
  • /debug via Debug
  • /dg via SignedDigestAndUnsignedPkcs7Path
  • /di via GenerateSignature
  • /dlib via AuthenticodeDigestSignLibDll
  • /dmdf via AuthenticodeDigestSignPassUnmodified
  • /ds via SignDigestOnly
  • /du via Url
  • /dxml via XmlFile
  • /f via File
  • /fd via FileDigestAlgorithm
  • /force via Force
  • /i via Issuer
  • /itos via IntentToSealAttribute
  • /kc via KeyContainer
  • /n via SigningSubjectName
  • /nosealwarn via NoSealWarn
  • /nph via SuppressPageHashes
  • /p via Password
  • /p7 via ContentFileToPkcs7
  • /p7ce via SignedContentMethod
  • /p7co via SignedContentIdentifier
  • /ph via PageHashes
  • /q via Quiet
  • /r via RootSubjectName
  • /rmc via RelaxedMarkerCheck
  • /s via Store
  • /sa via AuthenticatedAttributes
  • /seal via SealingSignature
  • /sha1 via Sha1Thumbprint
  • /sm via MachineStore
  • /t via TimestampServerUrl
  • /td via TimestampServerDigestAlgorithm
  • /tr via Rfc3161TimestampServerUrl
  • /tseal via Rfc3161TimestampServerUrlSealed
  • /u via EnhancedKeyUsage
  • /uw via WindowsSystemComponentVerification
  • /v via Verbose
