Class KubernetesApplySettingsExtensions

Namespace
Nuke.Common.Tools.Kubernetes
Assembly
Nuke.Common.dll

Apply a configuration to a resource by filename or stdin. The resource name must be specified. This resource will be created if it doesn't exist yet. To use 'apply', always create the resource initially with either 'apply' or 'create --save-config'. JSON and YAML formats are accepted. Alpha Disclaimer: the --prune functionality is not yet complete. Do not use unless you are aware of what the current state is. See https://issues.k8s.io/34274.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • --all via All
  • --allow-missing-template-keys via AllowMissingTemplateKeys
  • --alsologtostderr via Alsologtostderr
  • --as via As
  • --as-group via AsGroup
  • --cache-dir via CacheDir
  • --cascade via Cascade
  • --certificate-authority via CertificateAuthority
  • --client-certificate via ClientCertificate
  • --client-key via ClientKey
  • --cluster via Cluster
  • --context via Context
  • --dry-run via DryRun
  • --filename via Filename
  • --force via Force
  • --grace-period via GracePeriod
  • --include-uninitialized via IncludeUninitialized
  • --insecure-skip-tls-verify via InsecureSkipTlsVerify
  • --kubeconfig via Kubeconfig
  • --log-backtrace-at via LogBacktraceAt
  • --log-dir via LogDir
  • --logtostderr via Logtostderr
  • --match-server-version via MatchServerVersion
  • --namespace via Namespace
  • --openapi-patch via OpenapiPatch
  • --output via Output
  • --overwrite via Overwrite
  • --prune via Prune
  • --prune-whitelist via PruneWhitelist
  • --record via Record
  • --recursive via Recursive
  • --request-timeout via RequestTimeout
  • --selector via Selector
  • --server via Server
  • --stderrthreshold via Stderrthreshold
  • --template via Template
  • --timeout via Timeout
  • --token via Token
  • --user via User
  • --v via V
  • --validate via Validate
  • --vmodule via Vmodule
  • --wait via Wait
