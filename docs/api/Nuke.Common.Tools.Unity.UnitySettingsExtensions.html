Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <customArguments> via CustomArguments
  • -accept-apiupdate via AcceptApiUpdate
  • -assetServerUpdate via AssetServerUpdate
  • -batchmode via BatchMode
  • -buildLinux32Player via BuildLinux32Player
  • -buildLinux64Player via BuildLinux64Player
  • -buildLinuxUniversalPlayer via BuildLinuxUniversalPlayer
  • -buildOSX64Player via BuildOSX64Player
  • -buildOSXPlayer via BuildOSXPlayer
  • -buildOSXUniversalPlayer via BuildOSXUniversalPlayer
  • -buildTarget via BuildTarget
  • -buildWindows64Player via BuildWindows64Player
  • -buildWindowsPlayer via BuildWindowsPlayer
  • -cacheServerIPAddress via CacheServerIPAddress
  • -createProject via CreateProject
  • -disable-assembly-updater via DisableAssemblyUpdater
  • -editorTestsCategories via EditorTestsCategories
  • -editorTestsFilter via EditorTestsFilter
  • -editorTestsResultFile via EditorTestsResultFile
  • -executeMethod via ExecuteMethod
  • -exportPackage via ExportPackage
  • -force-clamped via ForceClamped
  • -force-d3d11 via ForceD3d11
  • -force-device-index via ForceDeviceIndex
  • -force-gfx-metal via ForceGfxMetal
  • -force-glcore via ForceGLCore
  • -force-glcore via ForceGLCoreXY
  • -force-gles via ForceGLES
  • -force-gles via ForceGLESXY
  • -force-low-power-device via ForceLowPowerDevice
  • -importPackage via ImportPackage
  • -logFile via LogFile
  • -nographics via NoGraphics
  • -noUpm via NoUpm
  • -password via Password
  • -projectPath via ProjectPath
  • -quit via Quit
  • -runEditorTests via RunEditorTests
  • -serial via Serial
  • -setDefaultPlatformTextureFormat via DefaultPlatformTextureFormat
  • -silent-crashes via SilentCrashes
  • -stackTraceLogType via StackTraceLogType
  • -username via Username

DisableBatchMode<T>(T)

Run Unity in batch mode. This should always be used in conjunction with the other command line arguments, because it ensures no pop-up windows appear and eliminates the need for any human intervention. When an exception occurs during execution of the script code, the Asset server updates fail, or other operations that fail, Unity immediately exits with return code 1.

Note that in batch mode, Unity sends a minimal version of its log output to the console. However, the Log Files still contain the full log information. Opening a project in batch mode while the Editor has the same project open is not supported; only a single instance of Unity can run at a time. 
[Pure]
public static T DisableBatchMode<T>(this T o) where T : UnitySettings

DisableNoGraphics<T>(T)

When running in batch mode, do not initialize the graphics device at all. This makes it possible to run your automated workflows on machines that don't even have a GPU (automated workflows only work when you have a window in focus, otherwise you can't send simulated input commands). Please note that -nographics does not allow you to bake GI on OSX, since Enlighten requires GPU acceleration.

Execute the static method as soon as Unity is started, the project is open and after the optional Asset server update has been performed. This can be used to do tasks such as continous integration, performing Unit Tests, making builds or preparing data. To return an error from the command line process, either throw an exception which causes Unity to exit with return code 1, or call EditorApplication.Exit with a non-zero return code.To use ExecuteMethod, you need to place the enclosing script in an Editor folder. The method to be executed must be defined as static.

It is good practice to pass the -batchmode and -quit arguments as well, in order to quit Unity when done, if using this for automated activation of Unity. Please allow a few seconds before the license file is created, because Unity needs to communicate with the license server. Make sure that license file folder exists, and has appropriate permissions before running Unity with this argument. If activation fails, see the Editor.log for info.

ToggleBatchMode<T>(T)

Run Unity in batch mode. This should always be used in conjunction with the other command line arguments, because it ensures no pop-up windows appear and eliminates the need for any human intervention. When an exception occurs during execution of the script code, the Asset server updates fail, or other operations that fail, Unity immediately exits with return code 1.

Note that in batch mode, Unity sends a minimal version of its log output to the console. However, the Log Files still contain the full log information. Opening a project in batch mode while the Editor has the same project open is not supported; only a single instance of Unity can run at a time.

ToggleNoGraphics<T>(T)

When running in batch mode, do not initialize the graphics device at all. This makes it possible to run your automated workflows on machines that don't even have a GPU (automated workflows only work when you have a window in focus, otherwise you can't send simulated input commands). Please note that -nographics does not allow you to bake GI on OSX, since Enlighten requires GPU acceleration.
