Class NpmInstallSettingsExtensions

Namespace
Nuke.Common.Tools.Npm
Assembly
Nuke.Common.dll

Installs a package, and any packages that it depends on. If the package has a package-lock or shrinkwrap file, the installation of dependencies will be driven by that, with an npm-shrinkwrap.json taking precedence if both files exist. See package-lock.json and npm-shrinkwrap.

A package is:
  • a) A folder containing a program described by a package.json file
  • b) A gzipped tarball containing (b)
  • c) A url that resolves to (b)
  • d) a <name>@<version> that is published on the registry (see npm-registry) with (c)
  • e) a <name>@<tag> (see npm-dist-tag) that points to (d)
  • f) a <name> that has a "latest" tag satisfying (e)
  • g) a <git remote url> that resolves to (a)

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <packages> via Packages
  • --force via Force
  • --global via Global
  • --global-style via GlobalStyle
  • --ignore-scripts via IgnoreScripts
  • --legacy-bundling via LegacyBundling
  • --link via Link
  • --no-bin-links via NoBinLinks
  • --no-optional via NoOptional
  • --no-shrinkwrap via NoShrinkWrap
  • --nodedir via NodeDir
  • --only via Only
  • --production via Production
