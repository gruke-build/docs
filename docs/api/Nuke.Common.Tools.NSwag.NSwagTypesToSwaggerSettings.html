Namespace
Nuke.Common.Tools.NSwag

Nuke.Common.dll

The project combines the functionality of Swashbuckle (Swagger generation) and AutoRest (client generation) in one toolchain. This way a lot of incompatibilites can be avoided and features which are not well described by the Swagger specification or JSON Schema are better supported (e.g. inheritance, enum and reference handling). The NSwag project heavily uses NJsonSchema for .NET for JSON Schema handling and C#/TypeScript class/interface generation.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • /AllowReferencesWithProperties via AllowReferencesWithProperties
  • /Assembly via Assembly
  • /AssemblyConfig via AssemblyConfig
  • /ClassNames via ClassNames
  • /DefaultDictionaryValueReferenceTypeNullHandling via DefaultDictionaryValueReferenceTypeNullHandling
  • /DefaultEnumHandling via DefaultEnumHandling
  • /DefaultPropertyNameHandling via DefaultPropertyNameHandling
  • /DefaultReferenceTypeNullHandling via DefaultReferenceTypeNullHandling
  • /FlattenInheritanceHierarchy via FlattenInheritanceHierarchy
  • /GenerateKnownTypes via GenerateKnownTypes
  • /GenerateXmlObjects via GenerateXmlObjects
  • /IgnoreObsoleteProperties via IgnoreObsoleteProperties
  • /Output via Output
  • /OutputType via OutputType
  • /ReferencePaths via ReferencePaths
  • /UseNuGetCache via UseNuGetCache
