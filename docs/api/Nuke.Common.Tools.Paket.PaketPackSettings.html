Paket is a dependency manager for .NET and mono projects, which is designed to work well with NuGet packages and also enables referencing files directly from Git repositories or any HTTP resource. It enables precise and predictable control over what packages the projects within your application reference.

If you want to learn how to use Paket then read the Getting started tutorial and take a look at the FAQs.

If you are already using NuGet for package management in your solution then you can learn about the upgrade process in the convert from NuGet section.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • <outputDirectory> via OutputDirectory
  • --build-config via BuildConfiguration
  • --build-platform via BuildPlatform
  • --exclude via Exclude
  • --from-bootstrapper via FromBootstrapper
  • --include-referenced-projects via IncludeReferencedProjects
  • --lock-dependencies via LockDependencies
  • --log-file via LogFile
  • --minimum-from-lock-file via MinimumFromLockFile
  • --pin-project-references via PinProjectReferences
  • --project-url via ProjectUrl
  • --release-notes via ReleaseNotes
  • --silent via Silent
  • --specific-version via SpecificVersions
  • --symbols via Symbols
  • --template via TemplateFile
  • --verbose via Verbose
  • --version via PackageVersion
