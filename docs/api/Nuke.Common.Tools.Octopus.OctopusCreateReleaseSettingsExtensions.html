Class OctopusCreateReleaseSettingsExtensions

Namespace
Nuke.Common.Tools.Octopus
Assembly
Nuke.Common.dll

The Octo.exe create-release can be used to automate the creation of releases. This allows you to easily integrate Octopus with other continuous integration servers.

For more details, visit the official website.

Remarks

This is a CLI wrapper with fluent API that allows to modify the following arguments:

  • --apiKey via ApiKey
  • --cancelontimeout via CancelOnTimeout
  • --channel via Channel
  • --configFile via ConfigFile
  • --debug via Debug
  • --deployat via DeployAt
  • --deploymentchecksleepcycle via DeploymentCheckSleepCycle
  • --deploymenttimeout via DeploymentTimeout
  • --deployto via DeployTo
  • --enableServiceMessages via EnableServiceMessages
  • --force via Force
  • --forcepackagedownload via ForcePackageDownload
  • --gitCommit via GitCommit
  • --gitRef via GitRef
  • --guidedfailure via GuidedFailure
  • --ignorechannelrules via IgnoreChannelRules
  • --ignoreexisting via IgnoreExisting
  • --ignoreSslErrors via IgnoreSslErrors
  • --logLevel via LogLevel
  • --norawlog via NoRawLog
  • --package via PackageVersions
  • --packageprerelease via PackagePrerelease
  • --packagesFolder via PackagesFolder
  • --packageversion via DefaultPackageVersion
  • --pass via Password
  • --progress via Progress
  • --project via Project
  • --proxy via Proxy
  • --proxyPass via ProxyPassword
  • --proxyUser via ProxyUsername
  • --rawlogfile via RawLogFile
  • --releasenotes via ReleaseNotes
  • --releasenotesfile via ReleaseNotesFile
  • --server via Server
  • --skip via Skip
  • --space via Space
  • --specificmachines via SpecificMachines
  • --tenant via Tenant
  • --tenanttag via TenantTag
  • --timeout via Timeout
  • --user via Username
  • --variable via Variables
  • --version via Version
  • --waitfordeployment via WaitForDeployment
  • --whatif via WhatIf
